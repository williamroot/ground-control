PHASE 02

O Service Desk de quem presta serviço quebrou.

ERR · 01

Você atende N clientes numa caixa só

Helpdesk genérico não foi feito para uma MSP que serve várias empresas distintas. Filas, SLA, marca e dados de clientes diferentes se misturam no mesmo balcão.

ERR · 02

O white-label é da ferramenta, não do seu cliente

No máximo você troca o logo do seu portal. Oferecer um portal com a marca de cada cliente seu que você atende? Não existe.

ERR · 03

Contratos de MSP não cabem na caixinha

Banco de horas, crédito compartilhado, franquia de deslocamento, glosa, ciclos de fechamento ≠ faturamento. Ferramenta genérica não modela isso.

ERR · 04

Seus dados moram na casa dos outros

As integrações que uma MSP precisa — Zabbix, PRTG, Microsoft 365, faturamento via Asaas/NFe — ou não existem ou são caça-níquel. A operação vira trabalho manual e propenso a erro.

CORREÇÃO DE ROTA

Ground Control foi desenhado para MSP: white-label por cliente final, multi-tenant de verdade, motor de contratos de MSP e faturamento automatizado — as features que as ferramentas genéricas não têm.

PHASE 03

Subsistemas a bordo.

Cada módulo é um subsistema independente do console. Tudo integrado, nada amarrado.

SYS·01

Contratos MSP

6 tipos: valor fechado, crédito, crédito compartilhado, banco de horas, SaaS/produto, por serviços. Ciclos de faturamento ≠ fechamento, reajuste por índice, renovação automática.

SYS·02

White-label por cliente

Cada cliente seu com subdomínio, logo, cores e e-mail próprios. Branding injetado no servidor antes do paint — zero flash genérico.

SYS·03

Multi-tenant blindado

Isolamento por linha (RLS no Postgres) + segregação de filas/ACLs. Pool compartilhado ou instância dedicada por cliente regulado.

SYS·04

Faturamento integrado

Consumo vira cobrança: Asaas, boleto, NFe. Glosa do cliente, reversal, conciliação. O ciclo fecha sozinho e gera a fatura.

SYS·05

Catálogo de serviços

Formulários dinâmicos por serviço, SLA por contrato, abertura guiada pelo cliente. Precificação automática amarrada ao contrato.

SYS·06

Dashboards & SLA

Telemetria por cliente: consumo do contrato, cumprimento de SLA, NPS, saldo de horas. O gestor do cliente vê o painel dele.

SYS·07

Integrações de campo

WhatsApp, e-mail, Microsoft 365/SSO, monitoramento Zabbix/PRTG abrindo ticket sozinho. API REST + webhooks HMAC para o resto.

SYS·08

Segurança & LGPD

Isolamento de dados por cliente (linha a linha), trilha de auditoria imutável e conformidade LGPD. Cada cliente só enxerga o que é dele — garantido na base.
